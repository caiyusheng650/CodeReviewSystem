## ⏱️ 时间估算方法

### 估算技术
#### 1. 规划扑克法 (Planning Poker)
```
步骤:
1. 产品负责人讲解需求
2. 每个开发者独立选择估算值(0,1,2,3,5,8,13,21,34)
3. 同时展示选择结果
4. 讨论最高和最低估算
5. 重新估算直到达成一致

适用场景: 新功能开发、复杂需求
优势: 团队参与度高，考虑全面
```

#### 2. 类比估算 (Analogous Estimation)
```
基于历史项目数据:
- 简单API开发: 2-4小时/接口
- 复杂业务逻辑: 1-3天/模块
- UI组件开发: 4-8小时/组件
- 测试用例编写: 30%开发时间
- Bug修复: 平均2小时/个

适用场景: 类似功能开发、维护任务
优势: 快速估算，基于经验
```

#### 3. 三点估算法 (Three-Point Estimation)
```
公式: (乐观 + 4×最可能 + 悲观) / 6

示例: 用户认证功能
- 乐观时间(O): 3天
- 最可能时间(M): 5天
- 悲观时间(P): 10天
- 估算结果: (3 + 4×5 + 10) / 6 = 5.5天

适用场景: 不确定性较高的任务
优势: 考虑风险因素，更准确
```

### 时间缓冲策略
#### 缓冲类型
- **项目级缓冲**: 总工期的15-20%
- **阶段级缓冲**: 每个阶段的10-15%
- **任务级缓冲**: 关键任务的20-25%
- **个人缓冲**: 个人工作量的10%

#### 缓冲分配
```
总工期: 7个月 (213天)
项目缓冲: 213 × 20% = 43天
可用工期: 213 - 43 = 170天

阶段缓冲分配:
- 阶段1: 60天 × 15% = 9天
- 阶段2: 75天 × 15% = 11天
- 阶段3: 60天 × 15% = 9天
- 阶段4: 18天 × 10% = 2天
```

#### 测评执行流程
```python
class EvaluationPipeline:
    def __init__(self):
        self.datasets = load_evaluation_datasets()
        self.metrics = define_evaluation_metrics()
        self.thresholds = set_performance_thresholds()
    
    def run_evaluation(self, dataset_name):
        """运行单个数据集测评"""
        print(f"🚀 开始测评数据集: {dataset_name}")
        
        # 1. 数据预处理
        data = self.preprocess_data(dataset_name)
        
        # 2. 模型推理
        predictions = self.model_predict(data)
        
        # 3. 结果评估
        results = self.evaluate_predictions(predictions, data.ground_truth)
        
        # 4. 指标计算
        metrics = self.calculate_metrics(results)
        
        # 5. 结果验证
        passed = self.validate_results(metrics)
        
        # 6. 生成报告
        report = self.generate_report(dataset_name, metrics, passed)
        
        return {
            'dataset': dataset_name,
            'metrics': metrics,
            'passed': passed,
            'report': report,
            'timestamp': datetime.now()
        }
    
    def comprehensive_evaluation(self):
        """运行全部5个数据集测评"""
        all_results = []
        
        for dataset in self.datasets:
            try:
                result = self.run_evaluation(dataset)
                all_results.append(result)
                
                if not result['passed']:
                    print(f"⚠️ 数据集 {dataset} 未通过测评")
                    self.analyze_failure(result)
                else:
                    print(f"✅ 数据集 {dataset} 通过测评")
                    
            except Exception as e:
                print(f"❌ 数据集 {dataset} 测评失败: {str(e)}")
                all_results.append({
                    'dataset': dataset,
                    'passed': False,
                    'error': str(e)
                })
        
        # 生成综合报告
        final_report = self.generate_comprehensive_report(all_results)
        return final_report
```

```python
class ContingencyPlan:
    def handle_evaluation_failure(self, dataset):
        """处理测评失败情况"""
        # 1. 立即分析问题根源
        root_cause = self.analyze_failure(dataset)
        
        # 2. 启动快速修复
        if self.can_quick_fix(root_cause):
            self.apply_quick_fix(dataset)
            return "快速修复，重新测评"
        
        # 3. 调整测评标准
        elif self.adjust_evaluation_criteria(dataset):
            return "调整标准，重新测评"
        
        # 4. 使用备用数据集
        else:
            self.switch_to_backup_dataset(dataset)
            return "启用备用数据集"
    
    def handle_video_delay(self, delay_days):
        """处理视频制作延期"""
        if delay_days <= 2:
            # 内部加班赶工
            self.crash_video_production()
        elif delay_days <= 5:
            # 简化制作要求
            self.simplify_video_requirements()
        else:
            # 启用外包方案
            self.activate_outsource_plan()
```

### 关键控制点
#### 时间预警机制
| 偏差程度 | 预警级别 | 响应措施 | 决策人 |
|----------|----------|----------|--------|
| **±5%** | 绿色 | 继续监控 | 项目经理 |
| **±10%** | 黄色 | 分析原因，调整计划 | 项目经理 |
| **±15%** | 橙色 | 启动应急预案 | 项目委员会 |
| **±20%** | 红色 | 重新评估项目 | 高级管理层 |

#### 应急措施
**进度超前时**:
- 提前启动下一阶段工作
- 增加功能优化和打磨时间
- 提前进行市场预热
- 增加测试和验证时间

**进度滞后时**:
- 调整任务优先级，聚焦核心功能
- 增加人力资源或外包支持
- 简化非核心功能
- 延长工作时间或加班

### 变更管理
#### 变更流程
```
变更申请 → 影响评估 → 审批决策 → 计划调整 → 团队通知 → 跟踪执行
    ↓         ↓         ↓         ↓         ↓         ↓
  书面申请   时间影响   项目经理   更新文档   全员通知   进度监控
  说明理由   资源影响   委员会     重新估算   明确责任   结果验证
```

#### 变更评估标准
- **必要性**: 变更是否必须
- **影响范围**: 对项目的影响程度
- **成本效益**: 投入产出比
- **风险等级**: 引入的新风险
- **可行性**: 技术和资源可行性
